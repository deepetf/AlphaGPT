# AlphaGPT V4.1 实施方案：动力学调优与工程加固 (低风险上线版)

## 目标 (Goal)
基于 V4.0 的反馈，通过引入**滑动窗口控制**、**多级成功定义**及**动态惩罚调节**，进一步消除熵控制器的“过敏反应”，精准治理长期坍缩。

## 方案差异对比 (V4.1 vs V4.0)

| 维度 | V4.0 (当前代码) | V4.1 (提议变更) | 理由 |
| :--- | :--- | :--- | :--- |
| **Pass 口径** | 单一 PASS (过回测+去重) | **Hard/Metric/Sim** 三级定义 | 将“语法合规”与“指标盈利”解耦，防止误触发 |
| **控制器输入** | 单步采样率 | **10步滑动窗口平均 (Rolling)** | 消除 Batch 间的随机抖动，平滑训练分配 |
| **恢复条件** | PR > 2% + Uniq > 80% | **PR > 2% 持续 10 步** | 移除过于苛刻的 Uniq 限制，提升恢复成功率 |
| **结构坍缩** | 仅根据最后一步比例 | **30步连续高位报警 (K=30, >90%)** | 捕捉趋势性坍缩，实施强力 Beta 锁定 |
| **低方差惩罚** | 固定值 (-6.5) | **触发式动态加深 (多步坍缩 -> -7.5)** | 避免前期过度加压，仅在顽固坍缩时给予重击 |
| **奖励饱和** | 仅 Gap 监控 | **Reward Std 移动标准差监控** | 实时监测奖励地形是否平坦化，预防搜索停滞 |

## 详细实施细节

### 1. 核心口径重定义 (Logic Layer)
- **HardPass**: 只要通过 `Struct`, `Exec`, `LowVar` 三道硬关（即 `status` 不属于该三类）。代表模型“学会了说话”。
- **MetricPass**: 在 HardPass 基础上通过回测指标（Sharpe, Active, Days 等）。代表模型“说的是有逻辑的好话”。
- **SimPass**: 在 MetricPass 基础上通过多样性去重。代表模型“说的是新鲜的好话”。
- **熵控制器输入**: 仅参考 **HardPassRate**。

### 2. 滑动窗口与控制器器 (Control Layer)
- 使用 `collections.deque(maxlen=10)` 维护 `HardPassRate` 序列。
- **触发规则**:
    - 一级：RollingPR < 1% -> $\beta$ 重置 0.04 (Max 0.06)。
    - 二级：1% ≤ RollingPR < 5% -> $\beta$ 线性微调 (+0.005)。
- **冷却逻辑**: 触发后锁定 10 步；恢复时检查 `StructFailShare` 均值是否未恶化。

### 3. 动态惩罚锁 (Penalty Layer)
- 若 `LowVarFailShare > 70%` 持续 50 步，动态将 `PENALTY_LOWVAR` 的权重系数从 1.0 提升至 1.15。

## 实施复杂度与合理性评估
- **合理性**: 极高。V4.1 把 RL 训练中常见的“均值复归”和“噪声滤除”引入了控制器，能显著减少 V4.0 可能出现的“永久高熵”现象。
- **实施难度**: 低到中。主要集中在 `engine.py` 的统计累加区域，无需修改底层 VM 或回测逻辑。

## 验收指标 (Refined)
1. **活性**: 后 200 步 `HardPassAbs_mean >= 1` 且 `HardPassRate_mean >= 0.3%`。
2. **迁移抑制**: 不存在 `StructFailShare > 90%` 持续 30 步。
3. **梯度成效**: `Gap` 均值/中位数下行趋势。

# 公式危险结构清单 (Dangerous Formula Structures)

**Version 2.0** | 更新时间: 2026-01-29

本文档记录了 AlphaGPT 进化搜索中应该避免的公式结构。这些结构会导致**分数坍缩**，即所有资产获得相同或极其相近的得分，使排名失去意义。

---

## 硬过滤规则 (直接拒绝)

### 相邻序列禁止

| 模式 | 风险描述 | 后果 |
|------|----------|------|
| `SIGN → LOG` | SIGN 输出 {-1, 0, 1}，LOG 将 -1/0 clamp 到常数 | 全资产同分 |
| `SIGN → SQRT` | SQRT 对 -1 无意义，只剩 {0, 1} | 信息离散化 |
| `ABS → SIGN` | ABS 后只有非负数，SIGN 只剩 {0, 1} | 信息离散化 |
| `CUT_HIGH → LOG` | CUT_HIGH 返回 -1e9，LOG 被 clamp | 异常值主导 |
| `NEG → SQRT` | 取负后开方被 clamp | 大面积常数 |
| `NEG → LOG` | 取负后对数被 clamp | 大面积常数 |

### 距离约束 (V2 新增)

| 规则 | 描述 | 示例 |
|------|------|------|
| **SIGN-LOG 距离 ≤ 2** | SIGN 和 LOG 间隔不超过 2 个 token 时拒绝 | `LOG → SIGN → LOG` ❌ |

### 末尾约束 (硬过滤)

| 规则 | 类型 | 描述 |
|------|------|------|
| 末尾是 `SIGN` | **硬过滤** | 最终输出被离散为 {-1, 0, 1}，排序退化 |
| 末尾连续离散算子 > 1 | **硬过滤** | 末尾连续出现多个 `SIGN`, `CUT_NEG`, `CUT_HIGH` |

## 软惩罚规则 (扣分但允许)

> **V2.2 Scale-Aware 惩罚体系** (考虑 `RobustConfig.SCALE=5.0`)
> - 高危 (数值爆炸): **-3.0** (等效 Sharpe 损失 -0.6)
> - 中危 (信息丢失): **-2.0** (等效 Sharpe 损失 -0.4)
> - 低危 (结构冗余): **-1.0** (等效 Sharpe 损失 -0.2)

### 高危: 数值爆炸风险 (-3.0)

| 模式 | 惩罚 | 风险描述 |
|------|------|----------|
| `SIGN → DIV` | -3.0 | 分母 {-1, 0, 1}，极易除零 |
| `CS_RANK → DIV` | -3.0 | 分母 [0,1]，接近 0 时爆炸 |

### 中危: 信息丢失风险 (-2.0)

| 模式 | 惩罚 | 风险描述 |
|------|------|----------|
| `CUT_NEG → LOG` | -2.0 | 大量 0 值导致 LOG 无区分度 |
| `CUT_NEG → DIV` | -2.0 | 分母可能为 0 |
| `CS_DEMEAN → LOG` | -2.0 | 均值=0，大量接近 0 的值 |
| `CS_DEMEAN → DIV` | -2.0 | 分母接近 0 |
| `SIGN ↔ CS_RANK` | -2.0 | 离散 + 排名 = 信息丢失 |
| `MIN → DIV` | -2.0 | 分母可能被截断接近 0 |
| 末尾 `CUT_NEG` / `CUT_HIGH` | -2.0 | 末尾截断导致信息丢失 |

### 低危: 结构冗余风险 (-1.0)

| 模式 | 惩罚 | 风险描述 |
|------|------|----------|
| `TS_STD5 → LOG` | -1.0 | 波动为 0 时 LOG 变常数 |
| `TS_DELTA → LOG/SQRT` | -1.0 | 差分可能为负，被 clamp |
| `TS_RET → LOG/SQRT` | -1.0 | 收益率可能为负 |
| `TS_BIAS5 → LOG/SQRT` | -1.0 | 乖离率可能为负 |
| `CS_RANK → LOG` | -1.0 | 输出 [0,1]，0 处 LOG 塌缩 |
| `IF_POS → LOG` | -1.0 | 分支可能输出 0 或负值 |
| `MIN → LOG` | -1.0 | MIN 可能取到被截断的 0/极小值 |

## 结构性惩罚

| 检查项 | 阈值 | 惩罚级别 | 惩罚值 |
|--------|------|----------|--------|
| 连续 `LOG` 数量 | > 2 | 低危 | 每超 1 个扣 1.0 分 |
| 连续 `TS_*` 数量 | > 2 | 低危 | 每超 1 个扣 1.0 分 |
| 离散算子总数 | > 3 | 低危 | 每超 1 个扣 1.0 分 |

## 算子分类参考

### 离散化算子
`SIGN`, `CUT_NEG`, `CUT_HIGH`

### 可能输出 0/负值的算子
`SIGN`, `CUT_NEG`, `CUT_HIGH`, `NEG`, `CS_DEMEAN`, `CS_RANK`, `TS_DELTA`, `TS_RET`, `TS_BIAS5`, `TS_STD5`
